# `docs/`

Documentation et rapports du projet.

## Dossiers importants

- `architecture/` : vue actuelle du systeme, contrats entre modules, registre des artefacts.
- `reports/module1/` : synthese du parsing.
- `reports/module2/` : audits de structuration grounded.
- `reports/mongodb/` : rapports import MongoDB.
- `reports/matching/` : Matching V3 et Decision Cards officielles.
- `reports/ml/` : experimentation ML, SHAP et feature importance.
- `reports/graph/` : Potential Graph et Neo4j Graph-RAG.
- `reports/demo/` : rapports de demonstration.
- `reports/retrieval/` : CrossEncoder et ablations retrieval.

## Regle

Les rapports dans `matching/v3/` et les rapports demo recents sont les plus importants pour la presentation. Les rapports ML et retrieval doivent etre presentes comme experimentaux.
